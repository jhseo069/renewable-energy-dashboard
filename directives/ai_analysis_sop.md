|------|--------|

## 분석 기능 범위

### Tab 1 — 조례 원문 분석 (`analyze_ordinance`)

| 항목 | 내용 |
|------|------|
| 입력 | 조례명, 기관명, 국가법령정보센터 원문 URL |
| 처리 | URL에서 HTML 원문 수집 → BeautifulSoup 텍스트 추출 → Gemini 분석 |
| 출력 | 마크다운 형식의 3단 구조 분석 결과 |
| 폴백 | 원문 수집 실패 시 조례명만으로 일반 분석 수행 |

### Tab 2 — 뉴스 동향 분석 (`analyze_news_trends`)

| 항목 | 내용 |
|------|------|
| 입력 | 뉴스 리스트 (제목, 요약, 언론사, 날짜), 키워드 |
| 처리 | 최대 20건 뉴스 텍스트 구성 → Gemini 분석 |
| 출력 | 마크다운 형식의 3단 구조 동향 분석 |
| 폴백 | 뉴스 0건이면 "분석할 뉴스가 없습니다." 반환 |

---

## 분석 결과 출력 형식 (시스템 프롬프트)

두 기능 모두 동일한 3단 구조:

```
1. 핵심 규제 내용 (또는 핵심 규제/정책 변화)
2. 사업개발 시 유의점
3. 대응 방안
```

## 기술 스택

| 항목 | 설정값 |
|------|--------|
| 패키지 | `google-generativeai>=0.8.0` |
| 온도 | `temperature=0.3` (사실 중심, 창의적 추론 최소화) |
| 최대 토큰 | `max_output_tokens=1024` |
| 원문 수집 타임아웃 | 15초 |
| 원문 최대 길이 | 8,000자 (Gemini 토큰 한도 고려) |

---

## 예외 처리 원칙

1. **`GEMINI_API_KEY` 미설정**: `ValueError` 발생 → UI에서 `st.error()` 표시
2. **원문 수집 실패 (404, 타임아웃)**: 빈 문자열 반환 → 조례명만으로 분석 (폴백)
3. **API 호출 실패**: `Exception` 발생 → UI에서 `st.error()` 표시
4. **빈 뉴스 리스트**: 조기 반환 ("분석할 뉴스가 없습니다.")

---

## UI 표시 원칙

- 각 조례 카드에 **"🤖 Gemini AI 분석"** 버튼 제공
- 각 뉴스 키워드 Expander에 **"🤖 Gemini AI 동향 분석"** 버튼 제공
- 분석 결과는 `st.session_state`에 저장 (페이지 리렌더 시 재분석 방지)
- 분석 결과는 Expander 또는 `st.markdown()` 으로 표시
- API 키 오류 시 명확한 에러 메시지 + 설정 방법 안내
